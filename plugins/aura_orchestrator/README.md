# AURA OS — Плагин: Мультиагентный Оркестратор

## Назначение
Режим «Эксперт» для глубокого анализа сложных вопросов. Использует иерархическую архитектуру из 5 контейнеров DeepSeek.

## Архитектура
```
Пользовательский запрос
        │
        ▼
┌──────────────────┐
│  Контейнер 1     │  Оркестратор — анализирует запрос и распределяет задачи
│  (Orchestrator)  │
└──────┬───────────┘
       │
       ├──► Контейнер 2a (Coordinator)   — план и архитектура
       ├──► Контейнер 2b (Researcher)    — глубокий анализ, альтернативы
       └──► Контейнер 2c (Developer)     — код и реализация
       │
       ▼
┌──────────────────┐
│  Дедупликатор    │  Локальная модель all-MiniLM-L6-v2
│  (локальный)     │  Удаляет повторы между ответами контейнеров 2
└──────┬───────────┘
       │
       ▼
┌──────────────────┐
│  Контейнер 3     │  Обозреватель — собирает единый ответ
│  (Reviewer)      │
└──────┬───────────┘
       │
       ▼
   Ответ пользователю
```

## Подключение

### 1. Зависимости
```bash
pip install sentence-transformers openai
```

### 2. Файлы плагина
```
plugins/aura_orchestrator/
├── aura_orchestrator.py   # основной код
└── README.md              # документация
```

### 3. Интеграция в AURA OS

**Telegram**: команда `/expert_chat [вопрос]`
- Без аргументов — переключает режим Эксперт/Аура
- С аргументами — отправляет вопрос эксперту, после ответа авто-возврат к Ауре

**Web-чат**: переключатель «Твоя Аура» / «Эксперт» в шапке чата
- «Твоя Аура» — обычный агент со всеми скиллами (память, календарь, погода, интернет)
- «Эксперт» — только оркестратор (без скиллов, чистый анализ)

**API**: `POST /api/chat/expert` с телом `{"text": "вопрос"}`

### 4. Ключ API
Используется `DEEPSEEK_API_KEY` из `.env` (основной ключ проекта).

### 5. Временные файлы
Создаются в `%TEMP%/aura_orchestrator/`, удаляются после каждого ответа.

## Режимы работы

| Режим | Агент | Скиллы | Назначение |
|-------|-------|--------|------------|
| Твоя Аура | AuraAgent | Все 19+ скиллов | Общение, календарь, погода, поиск |
| Эксперт | Orchestrator | Нет (чистый LLM) | Глубокий анализ, архитектура, код |

## Переключение
- После ответа Эксперта — автоматический возврат к «Твоя Аура»
- Аура может читать ответ Эксперта и работать с ним в рамках сессии
- История диалога сохраняется в памяти оркестратора между запросами (в рамках сессии)
